Artist Statement & Introduction

I approach digital media as a conversation between intention and accident. Over the semester I explored how algorithmic tools, intentional mark-making, and deliberate constraints can produce work that is both technically informed and emotionally resonant. I study how limitations — a restricted color palette, pixel grids, or the one-second GIF loop — can sharpen decisions and make composition and timing feel purposeful. My practice combines image-based experiments with code-driven sketches, and I enjoy shifting between rapid prototyping and careful refinement. Through iterative play I learned to read tools as collaborators: pixel manipulation taught me patience, masking exercises taught me precision, and p5.js sketches taught me how to make visuals respond to simple logic. This portfolio collects the artefacts of that process: thumbnails, process images, short reflections, and final pieces. Each project shows a facet of what I learned technically (selection, compositing, masking, animation, and coding) and creatively (visual rhythm, surprise, and narrative compression).

Pixelated

A set of artifacts exploring low-resolution aesthetics, color quantization, and grid-based composition.

Technical Notes

  • Tools: Photoshop, GIMP, ImageMagick (for batch downsampling), browser-based color-quantizer
  • Formats: PNG, optimized for web

Final Work

Reflection

Reducing images to a strict pixel grid taught me to think about which details actually read at small sizes. The constraint of a low-resolution canvas forced me to exaggerate key shapes and choose color contrasts that would survive aggressive quantization. Technically, I learned batch workflows in ImageMagick to automate downsampling and experimented with dithering options to create texture without adding complexity. Creatively, the project was about restraint — removing information until only the most important visual signals remained. I also tested different export sizes and compression settings to balance clarity and file size.

Bad Photoshop

An exercise in deliberate poor editing and playful collage — embracing odd proportions, mismatched lighting, and absurd compositions.

Technical Notes

  • Tools: Adobe Photoshop (or Photopea), layer blending modes, basic selection tools
  • Formats: JPEG/PNG — optimized GIF for animated variants

Process Documentation

Reflection

I intentionally broke conventional rules of compositing — ignoring consistent light source, scale, and perspective — to produce work that reads as humorous and uncanny. The biggest challenge was resisting the impulse to 'fix' things; instead I leaned into mismatches and used them as a compositional element. I learned to use quick layer masks and transform tools to juxtapose elements at odd scales. This project highlighted how deliberate errors can communicate voice and personality, and how technical lifts (like fast masking) let me rapidly iterate through more playful options.

Perfect Mask

Practicing precision in masking and compositing to create believable merges between foreground and background.

Technical Notes

  • Tools: Photoshop (Pen tool, Select & Mask), Photopea alternative, tablet for detailed selections
  • Techniques: feathering, edge refining, color-matching with adjustment layers

Process Documentation

Reflection

Masking is a deceptively complex skill: small edge artifacts can break the illusion of a composite. I practiced multiple selection methods and learned when to trade speed for precision (quick extract vs. pen tool). Color matching with curves and selective color adjustments was essential to sell the subject in a new environment. A key challenge was dealing with hair and semi-transparent edges; using layered masks and painting subtle density changes helped greatly. This project improved my patience and attention to micro-detail in compositing.

Animated GIFs

Short looping animations exploring timing, economy of motion, and visual rhythm.

Technical Notes

  • Tools: Photoshop timeline, After Effects (for complex easing), ezGIF or ImageMagick for optimization
  • Export: GIF optimized with limited palette and frame disposal settings

Process Documentation

Reflection

Making effective GIFs meant thinking in loops and negative space: what motion repeats, and what surprises on each cycle. I practiced paring down motion so that the loop reads cleanly at a glance. A technical hurdle was balancing frame rate, length, and palette size to keep file sizes small while maintaining visual clarity. I used dithering selectively to suggest color depth and tested different frame disposal methods to create smooth transitions. The project sharpened my sense of timing and compressed storytelling.

Reflection

Glitching taught me to accept unpredictability as a generative force. I learned safe workflows for corrupting copies, and how to curate accidental results into compositions. p5.js Landscape & Self Portrait

Generative landscape and a code-driven self-portrait using p5.js.

Technical Notes

  • Tools: p5.js, VS Code, Chrome/Firefox for testing
  • Techniques: noise(), perlin noise-based terrain, simple particle systems, color interpolation

Process Documentation

Reflection

Working in code required translating visual intuition into rules. Building the landscape was iterative: I tuned noise scales, palette interpolations, and layering to create depth. The self-portrait used simple shapes and mapped facial features to variables to allow generative variation. Challenges included debugging animation stutter and managing canvas resizing for different screens. The biggest learning was that small changes to parameters can dramatically shift mood; documenting those parameter choices helped me reproduce pleasing variations.

What If

A speculative project: reimagining a familiar image with alternate constraints.

Technical Notes

  • Tools: Photoshop, p5.js for variant generation, export to PNG
  • Approach: apply a single constraint (e.g., single hue, extreme aspect ratio, or motion-only edits) and iterate

Process Documentation

Reflection

What If asked me to reframe an image by applying a strict constraint. This made composition decisions clearer: once a single rule was in place, I could quickly reject or accept variations. I learned to use small scripts to batch-produce variants (useful when testing many possibilities) and to curate the strongest outcomes into a final series. The creative choice to work within a limit helped me discover surprising visual solutions that would not have emerged from an unconstrained approach.
